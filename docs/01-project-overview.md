# Project Overview

## Mission

Build a portable Linux cyberdeck while developing a practical understanding of modern laptop hardware architecture.

This project is intentionally documentation-first. Every component removed, tested, or modified should first be understood well enough to explain its purpose.

---

## Learning Goals

By the end of this project I should be able to:

- Identify the major subsystems of a laptop.
- Explain how power flows through the system.
- Explain how firmware starts a computer.
- Describe how Linux discovers and communicates with hardware.
- Confidently disassemble and reassemble a laptop.
- Troubleshoot common hardware failures.
- Design a portable enclosure around existing hardware.

---

## Engineering Principles

### Learn First

Avoid changing hardware until its purpose is understood.

### Incremental Progress

Each session should accomplish one clear objective.

### Documentation Is Part of the Build

Photos, notes, and diagrams are considered project artifacts—not optional extras.

### Preserve Optionality

Whenever possible, avoid irreversible modifications until the design has been validated.

---

## Definition of Success

The project is successful if:

- I understand the hardware better than when I started.
- Every major subsystem has been documented.
- The final device is repairable.
- The documentation would help another engineer reproduce the build.
